## UNION이란?

SQL에서 UNION이란 여러 SELETE문 결과를 하나로 합칠 때 사용하는 집합 연산자이다.
UNION의 종류로는 UNION, UNION ALL이 있다.

## UNION (중복 제거)

UNION은 여러 쿼리의 결과를 합친 후 종복된 행을 제거한다.
내부적으로 중복을 찾기위해 결과 세트를 정렬하는 과정을 거친다.
중복 체크와 정렬 작업 때문에 UNION ALL보다 속도가 느리다.

### 예시
```sql
SELECT 도서명 FROM Library_Books WHERE 도서명 LIKE '%SQL%'

UNION

SELECT 도서명 FROM Major_Books WHERE 도서명 LIKE '%SQL%';
```
다음은 UNION을 사용하는 예시이다.
Library_Books 테이블과 Major_Books테이블의 도서명 컬럼에서 SQL 문구를 찾아 하나로 합치고 중복을 제거하는 예시이다.

## UNION ALL (중복 포함)

UNION ALL은 중복을 포함하여 각 쿼리의 결과를 하나로 합친다.
별도의 정렬이나 중복체크 과정이 없어 처리 속도가 매우 빠르다.

### 예시

```sql
SELECT id, name FROM Class_A
UNION ALL
SELECT id, name FROM Class_B;
```
다음은 UNION ALL 예시이다.  
A반의 학생과 B반의 학생을 합쳐 전체 학생을 조회하고자 하는 예시이다. 

## 중요

> 컬럼 갯수가 같아야 한다.

## 정리

| 구분        | UNION                   | UNION ALL                 |
| ----------- | ----------------------- | ------------------------- |
| 결과물      | 중복 없는 고유한 행     | 중복을 포함한 모든 행     |
| 정렬 과정   | 발생함 (중복 제거 목적) | 발생하지 않음             |
| 속도        | 상대적으로 느림         | 빠름                      |
| 데이터 크기 | 결과 세트가 압축됨      | 모든 데이터가 합쳐져서 큼 |
