## 스칼라 서브쿼리(Scalar Subquery) 란?

**스칼라 서브쿼리**란 주로 **SELECT 절**에 사용되어 컬럼처럼 값을 반환하는 서브쿼리 이다.   
'스칼라(Scalar)'라는 단어의 뜻처럼 **단 하나의 값(1행 1열)**만을 결과로 내보내야 한다.



## 작성 예시
```sql
SELECT 
    name, 
    salary,
    -- 부서 테이블에서 부서명을 가져와 출력하는 스칼라 서브쿼리
    (SELECT dept_name 
     FROM departments d 
     WHERE d.id = e.department_id) AS dept_name
FROM employees e;
```

## 주요 특징
- 단일 행 반환: 서브쿼리 결과가 2행 이상이면 에러가 발생한다. (반드시 1행 1열)
- 일치 데이터 없음: 조건에 맞는 데이터가 서브쿼리에 없으면 NULL을 반환한다.
- 성능: 메인 쿼리의 데이터 건수만큼 서브쿼리가 반복 실행되므로 데이터가 너무 많을 때는 JOIN을 사용하는 것이 더 유리할 수 있다.

## 작성 규칙
- 반드시 괄호 ()로 감싸야 한다.
- 서브쿼리 안에서 메인 쿼리의 컬럼을 참조하여 데이터를 필터링할 수 있다.
- 결과 컬럼에 이름을 붙이기 위해 보통 AS(별칭)와 함께 사용한다.

> 스칼라 서브쿼리는 코드의 가독성을 높여주지만 대량의 데이터를 조회할 때는 실행 횟수를 반드시 고려해야 한다.
