## SQL WINDOW 함수

WINDOW 함수란 행과 행 간을 비교하거나 연산을 정의하기 위한 함수이다.
분석함수 또는 순위함수라고도 한다.
서브쿼리 등에서도 사용이 가능하다.

### 기본 사용법

```sql
SELECT WINDOW_FUNCTION (ARGUMENTS) OVER([PARTITION BY 컬럼] [ORDER BY 컬럼] [WINDOWING 절])
FROM 테이블명;
```

> over 문 필수!  
{: .prompt-tip }

## WINDOW FUNCTION 종류

| 구분                 | 함수                                            |
| -------------------- | ----------------------------------------------- |
| 순위함수             | RANK, DENSE_RANK, ROW_NUMBER                    |
| 일반 집계 함수       | SUM, MIN, MAX. AVG. COUNT                       |
| 그룹 내 행 순서 함수 | FIRST_VALUE, LAST_VALUE, LSG, LEAD              |
| 그룹 내 비율 함수    | RATIO_TO_REPORT, PERCENT_RANK, CUME_DIST, NTILE |

여러가지가 있는데 오늘은 순위함수에 대해 공부해보자.

## RANK (순위 함수)

| STORE_NAME (지점명) | REGION (지역) | SALES (매출) |
| ------------------- | ------------- | ------------ |
| 강남점              | 서울          | 500          |
| 홍대점              | 서울          | 700          |
| 해운대점            | 부산          | 600          |
| 서면점              | 부산          | 400          |
| 종로점              | 서울          | 300          |

각 지점의 매출 데이터 테이블이다.  
이를 사용하여 각 지역별 어떤 지점이 매출이 가장 높은지 순위를 매겨야 한다고 하였을 때

```sql
SELECT REGION, STORE_NAME, SALES,
       RANK() OVER (PARTITION BY REGION ORDER BY SALES DESC) AS REGION_RANK
  FROM CONVENIENCE_STORE;
```
![image](/assets/img/sqlRank.png)

이런식으로 쿼리문을 작성할 수 있다.

- `PARTITION BY`를 사용하여 각 지역별로 그룹화 하여 매출의 순위를 매길 수 있다.
- `ORDER BY`를 사용하여 순위를 매길 기준과 그 기준의 방향 (DESC -> 내림차순, ASC -> 오름차순)을 정할 수 있다.

또는

만약 전체 데이터의 순위를 원하는 경우 PARTITION BY를 사용하지 않으면 전체 데이터의 순위를 메길 수 있다.


```sql
SELECT REGION, STORE_NAME, SALES,
       RANK() OVER (ORDER BY SALES DESC) AS REGION_RANK
  FROM CONVENIENCE_STORE;
```
![image](/assets/img/sqlRank2.png)

> 동일한 값이 있을 경우 같은 순위를 부여하고 그다음 순위는 중복된 개수만큼 건너뛰고 매겨짐에 주의하자.  
예시: 1등이 2명이라면, 1, 1, 3, 4 이런식으로 순위가 매겨진다.  
{: .prompt-tip }


## DENSE_RANK (순위 함수 2)
RANK함수 중에 DENSE_RANK 함수도 있다.  
이 함수는 기존 RANK 함수와 비슷하지만 순위를 매기는 방식에 차이가 있다.
  
- RANK 함수는 위에서 설명한 듯 동일한 값이 있을 경우 그다음 순위는 중복의 개수만큼 건너뛰고 매겨져었다.
- **하지만 DENSE_RANK는 중복된 순위가 있다면 그 다음 순위는 중복 여부와 상관 없이 바로 다음 수가 매겨진다.**

같은 테이블에서 SQL 쿼리문을 작성해보겠다.
```sql
SELECT REGION, STORE_NAME, SALES,
       DENSE_RANK() OVER (ORDER BY SALES DESC) AS REGION_RANK
  FROM CONVENIENCE_STORE;
```
![image](/assets/img/sqldenserank.png)

위 결과와 같이 1위가 연속 2회 중복이어도 그다음부터는 2, 3, 4씩 다음 수 가 카운트 된다.
