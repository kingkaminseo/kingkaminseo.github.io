## SQL 인덱스란?

SQL 인덱스(Index)는 데이터베이스 테이블의 검색 속도를 향상시키기 위해 특정 컬럼의 데이터를 정렬하여 별도의 구조로 저장한 색인이다.


인덱스를 활용하면 전체 테이블을 스캔하지 않고도 특정 레코드의 위치를 빠르게 찾아낼 수 있어 SELECT 쿼리 성능을 극대화한다.

## 인덱스의 트레이드 오프
- 장점: SELECT 검색 속도 향상, ORDER BY 및 GROUP BY 작업 최적화, 정렬된 데이터 구조 활용.

- 단점: 추가적인 저장 공간 필요, 데이터 삽입(INSERT), 갱신(UPDATE), 삭제(DELETE) 시 인덱스도 함께 업데이트되어야 하므로 쓰기 성능 저하 발생 가능.


## 핵심 원리: 정렬과 탐색 (Sorting & Search)
인덱스의 핵심은 데이터를 특정 기준에 따라 정렬된 상태로 유지한다는 점에 있다. 이는 단순히 데이터를 모아두는 것이 아니라, 검색에 최적화된 형태로 가공하여 저장하는 것을 의미한다.

1. 정렬(Sorting): 무질서 속의 질서
데이터베이스의 실제 데이터 저장소(Heap)는 데이터가 삽입되는 순서대로 쌓이기 때문에 매우 무질서하다. 하지만 인덱스는 특정 컬럼을 기준으로 **미리 정렬(Pre-sorting)**을 해둔다.

왜 정렬하는가? 데이터가 정렬되어 있어야만 전체를 다 보지 않고도 내가 원하는 데이터가 "어디쯤에 있을지" 예측할 수 있기 때문이다.


이 부분을 공부하면서 B Tree란 부분의 언급이 많았다.
