## 정수 타입의 종류와 크기

| 타입명    | 용량 (Byte) | 범위 (Signed 기준) | 추천 용도                            |
| --------- | ----------- | ------------------ | ------------------------------------ |
| TINYINT   | 1           | -128 ~ 127         | "나이 , 성별 코드 , 상태값(0, 1)"    |
| SMALLINT  | 2           | 약 -3.2만 ~ 3.2만  | "카테고리 번호 , 소규모 인원수"      |
| MEDIUMINT | 3           | 약 -838만 ~ 838만  | "중간 규모 통계 , 우편번호"          |
| INT       | 4           | 약 -21억 ~ 21억    | "가장 일반적인 ID , 게시글 번호"     |
| BIGINT    | 8           | 약 -900경 ~ 900경  | "대규모 결제 로그 , 전 세계 유저 ID" |

## 5가지로 나눠서 사용하는 이유

그냥 다 정수형은 BIGINT쓰면 되는거 아닌가 하겠지만 이렇게 5가지로 나눠 사용하는데에는 크게 2가지 이유가 있다.

1. 저장 공간 최적화
   - TINYINT 사용 시 약 10MB
   - BIGINT 사용 시 약 80MB
   - TINYINT와 BIGINT는 무려 약 8배 정도의 용량차이가 발생하며 이능 서버 성능, 비용과 직결된다.
2. 처리 성능
   - 데이터 타입이 작을수록 인덱스 크기도 줄어들어 검색 속도가 빨라진다.
   - I/O 효율: 한번에 디스크 읽기에 더 많은 행을 담을 수 있어 입출력 효율이 빨라진다.

## 범위 2배 늘리기 (UNSIGNED)

정수형으로 사용해야할 데이터에서 음수의 사용이 불필요하다면 UNSIGNED 옵션을 사용하여
음수 영역을 제거하고 양수 영역을 2배 늘릴 수 있습니다.
TINYINT예를들어

- 기존 범위: -128 ~ 127
- UNSIGNED 사용 후 범위: 0 ~ 255

```sql
CREATE TABLE users (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    age TINYINT UNSIGNED
);

```

## 추가

### INT(10)이란

- INT 10의 10은 저장한계가 아니라 표현 형식이다.  
  실제로 INT를 사용하여서 약 -21억 ~ 21억까지 저장되는게 맞고 10의 자리를 미리 할당하는 것이다.

> 최신 표준: MySQL 8.0.17 버전 이후부터는 이 숫자 표시 방식이 Deprecated   (권장되지 않음) 되었으므로,   
그냥 `INT`라고만 쓰는 것이 현재의 표준이다.  
{: .prompt-tip }

### ZEROFILL

- 숫자의 빈자리를 0으로 채워서 출력해주는 옵션입니다.  
  INT(N)에서 지정한 숫자 N만큼 자릿수를 맞추는 역할을 합니다.

> 권장 사항: 현재는 DB에서 보여주는 형식을 관리하기보다,   
애플리케이션 코드(JS, Java 등)에서 출력 형식을 맞추는 방식을 권장합니다.  
{: .prompt-tip }
