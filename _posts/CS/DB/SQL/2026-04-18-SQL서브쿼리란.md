## 서브쿼리 란?

서브쿼리(subquery)란 다른 쿼리 내부에 포함되어있는 SELECT문을 말한다.

- 외부 쿼리 (Main Query): 서브쿼리를 감싸고 있는 전체 쿼리이다.
- 내부 쿼리 (Sub Query): 괄호 () 안에 포함된 하위 쿼리다.
  - 서브쿼리가 먼저 실행되어 결과를 반환하면, 외부 쿼리가 그 값을 넘겨받아 최종 결과를 낸다.

**작성 예시**
```sql
SELECT name, salary
FROM employees
WHERE salary > (
    SELECT AVG(salary)
    FROM employees
);
```
- 반드시 괄호 ()로 감싸야 한다
- 서브쿼리는 보통 비교연산자의 오른쪽에 위치한다.
- 서브쿼리 내에서는 메인쿼리의 컬럼을 참조할 수 있지만
메인 쿼리에서 서브쿼리의 컬럼을 직접 사용은 불가하다.



## 서브쿼리 장점
- 서브쿼리는 복잡한 JOIN이나 UNION과 같은 동작을 수행할 수 있는 또 다른 방법을 제공
- 서브쿼리는 쿼리를 구조화시키므로, 쿼리의 각 부분을 명확히 구분할 수 있게 해준다.
- 서브쿼리는 복잡한 JOIN이나 UNION 보다 좀 더 읽기 편함

## 중첩 서브쿼리 (Nested Subquery)
- Where 문에 사용하는 서브쿼리이다.
```sql
SELECT name
FROM employees
WHERE department_id = (
    SELECT id 
    FROM departments 
    WHERE dept_name = '마케팅'
);
```
다음처럼 서브쿼리가 외부 쿼리에 특정 조건 값을 전달하는 역할을 한다.


## 인라인 뷰 (Inline View)
- FROM문에 사용하는 서브쿼리이다. 
- 서브쿼리의 결과는 하나의 임시 테이블처럼 작동한다.
```sql
SELECT super_student.name
FROM (
    SELECT name, score
    FROM students
    WHERE score >= 90
) AS super_student
```

> 인라인 뷰를 사용할 때는 반드시 AS(별칭)을 사용하여 외부 쿼리에 참조를 해야한다.
