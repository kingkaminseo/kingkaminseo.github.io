## SQL VIEW
SQL에서 VIEW란 물리적으로 데이터를 저장하지 않고 호출될 때마다 정의된 SQL을 실행하는 '가상 테이블' 역할을 한다.

## VIEW문 생성 및 조회 예시
```sql
CREATE VIEW v_active_users AS
SELECT id, name, email
FROM users
WHERE status = 'active';
```
```sql
SELECT * FROM v_active_users;
```
## 핵심개념
- VIEW 안에는 실제 데이터가 들어있지 않다. 
- '어떤 쿼리를 실행해라!'라는 명령문만 저장되어 있다.  
- 원본 데이터가 바뀌면 VIEW를 통해 보이는 데이터도 실시간으로 변경된다.

## Trade-off
### 장점
- 재사용성
    - 복잡한데 재사용성이 있는 쿼리문을 VIEW문으로 아주 간단하게 만들어준다.
- 보안
노출되면 안되는 컬럼을 제외하고 VIEW로 만들어 VIEW에 접근권한을 주어 중요한 컬럼들을 간단하게 숨길 수 있다.

### 단점
- 인덱스 사용불가
- VIEW는 조회용이라 원본 데이터의 수정(INSERT, UPDATE)에 제약이 있다.
