## 웹 쿠키란?
HTTP(웹 쿠키 또는 브라우저 쿠키) 쿠키란 웹서버가 웹 브라우저로 전송하는 작은 데이터 조각이다.
브라우저는 이 쿠키를 웹에 저장해두었다가 동일한 웹서버에 요청시 이 쿠키를 사용하여 

## 사용이유 (http의 특성)
HTTP 프로토콜은 **Connectionless(비연결성)** 과 **Stateless(무상태성)** 이라는 특징을 가진다.  
즉, 클라이언트가 서버에 요청을 보내고 응답을 받으면, 해당 연결은 바로 종료된다.  
또한 각 요청은 서로 독립적이기 때문에 이전 요청의 상태 정보가 유지되지 않는다.  
  
이러한 특성 때문에 서버는 클라이언트의 이전 상태를 알 수 없으며   
클라이언트 역시 이전에 맺었던 연결 정보를 유지하지 않는다.  

예를 들어, 로그인 기능을 생각해보면  
사용자가 한 번 로그인 한 이후의 요청에서는  
서버가 해당 사용자가 이미 인증된 사용자라는 사실을 자동으로 알 수 없다.  

다음 예시와 같이 로그인 상태를 유지하기 위해서는  
쿠키, 세션, 토큰(JWT)과 같은  별도의 상태 관리 방법이 필요하다.  

## 쿠키사용 용도
### 사용자 세션

쿠키는 사용자의 로그인 상태를 유지하는 데 활용된다.
사용자가 로그인하면 서버는 세션 ID를 생성하고 이를 쿠키에 담아 클라이언트에 저장한다.
이후 클라이언트는 요청을 보낼 때마다 해당 쿠키를 함께 전송하고,
서버는 이를 통해 사용자를 식별하고 인증 상태를 유지할 수 있다.

### 개인화

쿠키는 사용자 맞춤 설정을 저장하는 데 사용된다.
예를 들어 다크 모드 여부, 언어 설정, 최근 본 상품 등과 같은 정보를 저장하여
사용자에게 더 편리한 경험을 제공할 수 있다.

### 추적

쿠키는 사용자의 행동을 분석하기 위한 용도로도 활용된다.
웹사이트 방문 기록, 클릭 패턴 등을 추적하여
서비스 개선이나 맞춤형 광고 제공에 사용될 수 있다.


## 쿠키의 다른 유형은?
### 세션 쿠키
세션 쿠키는 웹사이트에서 사용자의 세션을 추적하기 위해 사용된다.
이 쿠키는 브라우저에 임시로 저장되며, 사용자가 로그아웃하거나 브라우저를 종료하면 자동으로 삭제된다.
별도의 만료 시간이 설정되어 있지 않으며, 세션이 종료될 때 함께 제거된다.

### 영구 쿠키
영구 쿠키는 일정 기간 동안 브라우저에 저장되는 쿠키이다.
하루, 몇 주, 몇 달 등 미리 설정된 만료 시점까지 유지되며,
사용자의 재방문 시에도 정보를 유지할 수 있다는 특징이 있다.

### 인증 쿠키

인증 쿠키는 사용자의 로그인 상태를 유지하는 데 사용된다.
사용자가 로그인하면 생성되며, 서버는 쿠키에 포함된 식별자를 통해
해당 사용자의 세션을 식별하고 인증 정보를 확인한다.

### 추적 쿠키

추적 쿠키는 사용자의 활동을 분석하기 위해 사용된다.
사용자의 방문 기록, 클릭 패턴 등을 수집하여
서비스 개선이나 맞춤형 콘텐츠 및 광고 제공에 활용된다.

### 좀비 쿠키

좀비 쿠키는 사용자가 삭제하더라도 다시 생성되는 특성을 가진 쿠키이다.
일반적인 쿠키 저장소 외부에 백업 데이터를 보관하고 있다가 이를 통해 복원된다.
주로 광고 추적 등에 사용되며, 개인정보 보호 측면에서 문제가 될 수 있다.


## 쿠키 취약점
- Session hijacking

- XSS(Cross Site Scripting)

- CSRF(Cross-Site Request Forgery)

## 쿠키 보안속성
이러한 취약점을 사전에 방지하기 위해 쿠키의 보안 속성을 적절히 설정하여 보안을 강화할 수 있다.

### Secure
요청이 HTTPS와 같은 보안 채널을 통해 전송되는 경우에만 쿠키를 전송하도록 브라우저에 지시한다.   
이를 통해 암호화되지 않은 요청에서 쿠키가 전달되지 않도록 보호할 수 있다.

### HttpOnly
HttpOnly는 JavaScript와 같은 클라이언트 측 스크립트를 통해 쿠키에 액세스할 수 없도록 하여   
세션 유출과 같은 공격을 방지하는 데 사용된다. 

> XSS 공격을 전부 막을 수는 없지만 XSS 공격 벡터의 도달 범위를 제한할 수 있다.
