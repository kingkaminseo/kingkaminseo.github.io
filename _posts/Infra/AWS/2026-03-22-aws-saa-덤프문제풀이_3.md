## 문제 3-1
한 회사가 AWS에서 여러 Windows 워크로드를 실행합니다.
회사 직원은 두 개의 Amazon EC2 인스턴스에서 호스팅 되는 Windows 파일 공유를 사용합니다.
파일 공유는 서로 간에 데이터를 동기화하고 복제본을 유지합니다.
회사는 사용자가 현재 파일에 액세스 하는 방식을 보존하는 가용성이 높고 내구성이 뛰어난 스토리지 솔루션을 원합니다.
솔루션 설계자는 이러한 요구 사항을 충족하기 위해 무엇을 해야 합니까?
  

### 선택지
1. 모든 데이터를 Amazon S3로 마이그레이션합니다. 사용자가 파일에 액세스할 수 있도록 IAM 인증을 설정합니다.
2. Amazon S3 파일 게이트웨이를 설정합니다. 기존 EC2 인스턴스에 S3 File Gateway를 탑재합니다.
3. 다중 AZ 구성을 사용하여 파일 공유 환경을 Windows 파일 서버용 Amazon FSx로 확장합니다. 모든 데이터를 FSx for Windows File Server로 마이그레이션합니다.
4. 다중 AZ 구성을 사용하여 파일 공유 환경을 Amazon Elastic File System(Amazon EFS)으로 확장합니다. 모든 데이터를 Amazon EFS로 마이그레이션합니다.
  

정답: 3


    
### 용어 및 개념 정리 
- Amazon FSx for Windows File Server: 완전 관리형 원격 파일 서버 솔루션으로 Windows 네이티브 파일 시스템 기느을 제공한다.  
    - 데이터를 동기화 및 복제를 지원하고 여러 인스턴스에서 일관된 데이터 접근을 보장한다.
    - 높은 가용성과 내구성을 제공한다. (가용 영역(AZ)에서 자동으로 복제하여 데이터 손실 방지)

<mark>**Amazon FSx for Windows File Server는 Windows용 파일 저장소 서비스**</mark>
### 오답 이유

1. 모든 데이터를 Amazon S3로 마이그레이션합니다. 사용자가 파일에 액세스 할 수 있도록 IAM 인증을 설정합니다.
    - S3는 객체 스토리지로 전통적인 파일 시스템 접근 방식을 제공하지 않음. (요구사항 충족 X)
    - S3는 데이터 동기화 및 복제를 자동으로 지원하지 않음 (오버헤드 증가)

2. Amazon S3 파일 게이트웨이를 설정합니다. 기존 EC2 인스턴스에 S3 File Gateway를 탑재합니다.
    - S3 파일 게이트웨이 또한 객체 스토리지. (높은 가용성과 내구성을 제공하나 동기화 및 복제 기능이 제공되지 않아 오버헤드 증가)

3. O
4. 다중 AZ 구성을 사용하여 파일 공유 환경을 Amazon Elastic File System(Amazon EFS)으로 확장합니다. 모든 데이터를 Amazon EFS로 마이그레이션합니다.
    - **Amazon EFS**: 완전 관리형 NFS 파일 시스템
        - 리눅스 기반 워크로드에 적합. 
        - 고가용성 제공.
    - Amazon EFS는 리눅스 기반의 NFS 파일 시스템으로 설계되어 있어 Windows  파일 공유 요구사항을 충족하지 않음.



	
## 문제 3-2
회사는 최근 청구서에서 Amazon EC2 비용이 증가했음을 확인했습니다.  
청구 팀은 몇 가지 EC2 인스턴스에 대한 인스턴스 유형의 원치 않는 수직 확장을 발견했습니다.  
솔루션 아키텍트는 최근 2개월간의 EC2 비용을 비교하는 그래프를 생성하고 심층 분석을 수행하여 수직 확장의 근본 원인을 식별해야 합니다.  
솔루션 설계자는 최소한의 운영 오버헤드로 정보를 어떻게 생성해야 합니까?

### 선택지
1. AWS 예산을 사용하여 예산 보고서를 생성하고 인스턴스 유형에 따라 EC2 비용을 비교합니다.
2. 비용 탐색기의 세분화된 필터링 기능을 사용하여 인스턴스 유형에 따라 EC2 비용을 심층 분석합니다.
3. AWS Billing and Cost Management 대시보드의 그래프를 사용하여 지난 2개월 동안 인스턴스 유형을 기준으로 EC2 비용을 비교합니다.
4. AWS 비용 및 사용 보고서를 사용하여 보고서를 생성하고 Amazon S3 버킷으로 보냅니다. Amazon S3와 함께 Amazon QuickSight를 소스로 사용하여 인스턴스 유형을 기반으로 대화형 그래프를 생성합니다.

정답: 2


### 용어 및 개념 정리 
<mark>**최근 2개월간의 EC2 비용을 비교하는 그래프 생성 및 수직 확장의 근본 원인을 심층 분석**</mark>
- **AWS Cost Explorer**:   
AWS 비용과 사용량을 시각적으로 분석할 수 있는 도구입니다.  
Cost Explorer를 사용하면 최근 2개월간의 EC2 비용을 비교하는 그래프를 생성할 수 있습니다.
- **Amazon CloudWatch**:  
AWS 리소스와 애플리케이션의 모니터링 서비스를 제공하며, 지표 및 로그 데이터를 수집하고 시각화할 수 있습니다.  
CloudWatch를 사용하여 EC2 비용 관련 메트릭을 수집하고 대시보드를 통해 시각화할 수 있습니다.  
- **AWS Budgets**:   
비용 및 사용량 목표를 설정하고 모니터링할 수 있게 해 줍니다.  
비용이 예상치를 초과하면 알림을 받을 수 있습니다.  
이를 통해 비용 추이를 모니터링하고 분석할 수 있습니다.
- **AWS Trusted Advisor**:  
AWS 환경을 최적화하고 비용 절감, 보안 강화, 성능 향상을 위한 권장 사항을 제공합니다.  
Trusted Advisor를 사용하여 수직 확장의 원인을 분석할 수 있습니다.
- **AWS Cost and Usage Report**:  
AWS 계정의 비용 및 사용량 데이터를 세부적으로 제공합니다.  
이를 통해 수직 확장의 근본 원인을 분석할 수 있습니다.
- **Amazon CloudWatch Logs Insights**:  
CloudWatch Logs에서 수집된 로그 데이터를 쿼리하고 분석할 수 있는 도구입니다.  
이를 통해 EC2 인스턴스의 로그 데이터를 심층 분석하여 수직 확장의 원인을 찾을 수 있습니다.  
- **AWS Compute Optimizer**:  
AWS 리소스의 성능과 비용을 최적화하는 데 도움을 주는 서비스입니다.  
Compute Optimizer는 현재 인스턴스 사용 패턴을 분석하여 적절한 인스턴스 유형을 추천합니다.

### 오답 이유
1. AWS 예산을 사용하여 예산 보고서를 생성하고 인스턴스 유형에 따라 EC2 비용을 비교합니다.
    - AWS Budget은 비용 예측 및 알림이 초점으로 맞추어져 있다.

2. 비용 탐색기의 세분화된 필터링 기능을 사용하여 인스턴스 유형에 따라 EC2 비용을 심층 분석합니다.
    - 비용 탐색기 = (AWS Cost Explorer)
    - 요구사항 모두 만족
3. AWS 비용 및 사용 보고서를 사용하여 보고서를 생성하고 Amazon S3 버킷으로 보냅니다. Amazon S3와 함께 Amazon QuickSight를 소스로 사용하여 인스턴스 유형을 기반으로 대화형 그래프를 생성합니다.
    - 고급 분석 대시보드 X
    - 세분화된 분석 기능 부족
3. AWS 비용 및 사용 보고서를 사용하여 보고서를 생성하고 Amazon S3 버킷으로 보냅니다. Amazon S3와 함께 Amazon QuickSight를 소스로 사용하여 인스턴스 유형을 기반으로 대화형 그래프를 생성합니다.
    - AWS Cost and Usage Report는 가장 세부적인 비용 데이터를 제공하지만 설정이 복잡할 수 있음
    - 오버헤드 증가

## 문제 3-3
회사는 1주일 동안 지속되는 예정된 이벤트를 위해 특정 AWS 리전에 있는 3개의 특정 가용 영역에서 보장된 Amazon EC2 용량이 필요합니다.

회사는 EC2 용량을 보장하기 위해 무엇을 해야 합니까?

### 선택지
1. 필요한 리전을 지정하는 예약 인스턴스를 구매합니다.
2. 필요한 리전을 지정하는 온디맨드 용량 예약을 생성합니다.
3. 필요한 리전과 3개의 가용 영역을 지정하는 예약 인스턴스를 구입합니다.
4. 필요한 리전 및 세 개의 가용 영역을 지정하는 온디맨드 용량 예약을 생성합니다.
